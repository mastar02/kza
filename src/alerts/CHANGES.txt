src/alerts/alert_scheduler.py (395 líneas)
├─ Módulo: Sistema de scheduler para verificaciones periódicas
├─ Clases:
│  ├─ AlertScheduler: Controlador principal del scheduler
│  ├─ CheckType: Enum para tipos de verificación (SECURITY, PATTERN, DEVICE)
│  └─ CheckConfig: Dataclass para configuración de chequeos
├─ Métodos principales:
│  ├─ start(): Inicia el scheduler (bloquea hasta stop)
│  ├─ stop(): Detiene el scheduler
│  ├─ register_*_handler(): Registra handlers personalizados
│  ├─ set_check_enabled(): Control dinámico
│  ├─ set_check_interval(): Cambiar intervalos
│  └─ get_status(): Estado actual
└─ Características:
   ├─ Ejecución asincrónica en background
   ├─ 3 tipos de verificación con intervalos configurables
   ├─ Manejo robusto de errores
   └─ Sin dependencias externas

9. NOTAS DE DISEÑO
================================================================================

- Async-first: Todos los componentes principales son async
- Modular: Cada tipo de alerta es independiente
- Configurable: Parámetros ajustables en YAML
- Extensible: Fácil agregar nuevos tipos de verificación
- Sin dependencias: Solo stdlib y módulos del proyecto
- Error-tolerant: Continúa funcionando aunque haya errores
- Type-safe: Type hints completos en todo el código
- Well-documented: Docstrings, guías y ejemplos

10. COMPATIBILIDAD
================================================================================

Versión anterior: ✓ Compatible
- Todos los componentes existentes funcionan sin cambios
- Se pueden usar juntos sin conflictos
- La configuración de YAML es opcional (alerts.enabled = false por defecto)

Requiere:
- Python 3.7+ (async/await)
- asyncio
- logging (stdlib)
- yaml (ya en proyecto)

No requiere:
- Dependencias externas nuevas
- Cambios en Home Assistant
- Cambios en TTS service
